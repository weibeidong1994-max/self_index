## 结构化 Prompt 的力量

很多开发者把 Prompt 当作简单的查询字符串，但实际上，结构化的 Prompt 能够显著提升模型输出的稳定性和质量。一个经过验证的模板包括：角色定义（Role）、背景信息（Context）、任务描述（Task）、输出格式（Format）和约束条件（Constraints）。这种 RC-TFC 结构让模型明确知道"我是谁、我在什么场景下、我要做什么、怎么做"。

## Few-shot 示例的选取策略

在需要特定输出格式的场景中，Few-shot 示例几乎是必不可少的。但示例的质量比数量更重要。选取示例时，应该优先覆盖边界情况和易错场景，而不仅仅是理想路径。同时，示例之间应该保持风格一致，避免给模型传递矛盾的格式信号。实践中，3-5 个精心设计的示例往往比 10 个随意选取的示例效果更好。

## 迭代优化的方法论

Prompt 优化不是一次性工作，而是一个持续迭代的过程。建议建立 Prompt 版本管理系统，记录每次改动的动机和效果对比。A/B 测试是验证 Prompt 改进的有效手段——用相同的测试集分别跑新旧版本 Prompt，从准确率、格式合规率和用户满意度三个维度评估。记住，没有"完美"的 Prompt，只有"在当前场景下足够好"的 Prompt。
